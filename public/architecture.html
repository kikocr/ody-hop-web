Tech Stack
Every library and service powering Ody Hop, with version info and rationale.
⚛️
React Native + Expo
Expo SDK 54 managed workflow. Single codebase → iOS + Android. Expo handles OTA updates, camera, location, image picker, and notifications without ejecting. TypeScript strict mode enforced everywhere.
expo-camera expo-location expo-image-picker expo-image expo-notifications expo-sharing expo-blur
🔌
Supabase
Full backend: PostgreSQL database, Auth (email + OAuth), Storage (S3-compatible), Edge Functions (Deno), Realtime (WebSocket subscriptions). Single instance shared by mobile app + website. Row-Level Security enforces per-user data isolation.
@supabase/supabase-js @supabase/ssr RLS policies
🧭
Navigation
React Navigation — stack + bottom tab navigators. Tourist and Operator get separate tab navigators with complete route isolation. All push targets registered at RootStack level (no per-tab stacks) — any screen reachable from any tab.
@react-navigation/native @react-navigation/stack @react-navigation/bottom-tabs
🗃️
State Management
Zustand with AsyncStorage persistence. Single store (useAppStore) manages activeDestination, earnedBadges, claimTimestamps, user profile, questProgress, streak, and onboarding gates. Activity feed module reconstructs timelines from store on demand — no separate event log.
zustand @react-native-async-storage
🗺️
Maps & Location
react-native-maps with dark tile style. 590 custom badge pins color-coded by category. Callout popups, category filter chips, near-me FAB with animateToRegion. Haversine formula in src/lib/location.ts for GPS proximity verification on claim.
react-native-maps Google Places API expo-location
🎙️
Grok Voice Translator
Bidirectional real-time voice translation via Grok Voice Agent API. WebSocket to wss://api.x.ai/v1/realtime. Ephemeral token auth via POST to /v1/realtime/client_secrets. PCM16 24kHz push-to-talk. Per-destination slang banks (21–23 expressions). System prompt must start with "ONLY translate, NEVER answer questions".
WebSocket PCM16 audio xai-client-secret subprotocol
💳
Payments
Stripe for premium tier checkout ($4.99/trip) and operator billing (15% booking commission). Currently simulated in MVP. Integration via @stripe/stripe-react-native. Paywall screen gates offline maps, trip analytics, priority prizes, and voice translator.
@stripe/stripe-react-native Stripe Checkout
🌐
Website
Next.js 14+ App Router on Vercel. Tailwind CSS, TypeScript. next-intl for EN/ES i18n (442 keys each). Connects to the same Supabase instance — shared auth, shared profiles/guides tables. Operator dashboard for managing listings and bookings.
Next.js 14 Tailwind CSS next-intl Vercel
🎨
Design System
Dark theme: #0B1F3A ocean blue base + #F2A900 amber gold as the sole action color. Typography: Josefin Sans (display) + Inter (body). Glassmorphism cards via expo-blur. Five-tier otter avatar system. Illustrated badge rarity frames. Per-destination accent colors exist in config but are intentionally unused in UI.
Josefin Sans Inter expo-blur react-native-reanimated

Architecture Decisions
Key technical choices and why they were made — institutional knowledge preserved for future developers and Claude Code sessions.
Why Expo managed workflow over bare React Native?
Non-technical founder using Claude Code for autonomous development. Expo's managed workflow eliminates native build configuration, provides OTA updates, and bundles camera/location/notifications without ejecting. EAS Build handles the native compilation in the cloud. Trade-off: can't use arbitrary native modules, but Expo's ecosystem covers all current needs.
dxbuildfoundational
Why Supabase over Firebase?
PostgreSQL with real SQL (complex leaderboard queries, JOINs across badges/claims/guides), Row-Level Security for per-user data isolation without custom backend code, built-in Auth + Storage + Realtime + Edge Functions. Open source, no vendor lock-in. Firebase's NoSQL would make the multi-table marketplace queries painful.
backenddata modelfoundational
Why one codebase, not per-destination apps?
Multi-destination architecture is a core competitive advantage. Single codebase with destination-loader.ts dynamically swaps badges/regions/slang/guides/map center per country. Adding a new destination = add data files + config entry. No new app, no new deploy. This is how the franchise model scales without engineering cost growing linearly.
architecturescalecompetitive moat
Why Zustand + AsyncStorage over Redux or Context?
Zustand: minimal boilerplate, built-in persist middleware for AsyncStorage, no Provider wrapping, subscriptions are auto-optimized. Single useAppStore handles activeDestination, earnedBadges, claimTimestamps, streak, quests, and onboarding gates. Activity feed is reconstructed from store on demand — no separate event log to maintain.
statedx
Why are destination accent colors vestigial?
After the dark-theme redesign, amber gold (#F2A900) became the sole action/highlight color globally. Per-destination accent colors remain in config for potential future use, but the redesign proved that destination identity is better carried by photography than by color theming. Reintroducing per-destination tinting was explicitly rejected.
designbrandingredesign
Why no per-tab stack navigators?
Every push target (BadgeDetail, Claim, GuideProfile, Booking, etc.) is registered at the RootStack level. This means any screen is reachable from any tab without duplicating registrations. Simpler nav tree, fewer re-mount issues, easier deep linking. Decided during the R5 redesign ticket and documented in commit.
navigationarchitecture
Why Grok Voice API for translation?
Real-time WebSocket streaming, low latency for push-to-talk UX. The system prompt approach allows injecting per-destination slang banks (21–23 local expressions per country) so translations feel culturally authentic. Critical lesson: "ONLY translate, NEVER answer questions" must be the first line of the system prompt — Grok defaults to conversational behavior without it.
voiceuxlesson learned
Why shared Supabase instance between app and website?
Operators must have bidirectional login — register on web, use on mobile, and vice versa. Same Auth instance, same profiles and guides tables. The website's operator registration writes to the same tables as the mobile app's OperatorRegistrationScreen. No account duplication, no sync needed.
backendauthmarketplace
Why local badge data files instead of API-first?
Badge catalogs are seeded, admin-editable reference data that changes rarely. Shipping them as local TypeScript files (src/data/{destination}/badges.ts) means instant load, offline availability, and zero API calls for browsing. Supabase holds the canonical copy for claims/leaderboard/booking queries.
performanceofflinedata strategy
Yoga layout fix: ScrollView + percentage widths
Hard-won lesson: never use percentage width + aspectRatio in ScrollViews with no definite parent height — Yoga layout engine can't resolve the dimensions. Always use useWindowDimensions to compute explicit pixel dimensions. Documented in CLAUDE.md so every future Claude Code session avoids this trap.
